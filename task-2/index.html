Operation diagnosis

Copy page ⌄
GET /projects/{projectIdOrName}/operations/{operationId}/diagnosis Try it
Proposed endpoint (Task 2). Diagnose step — returns a structured, deterministic diagnosis of a single failed operation.

Returns a classified root cause, supporting evidence, and executable remedy for a failed operation — so the dashboard, a builder's tooling, and AI agents consume the same answer without re-implementing the logic. The natural-language summary is opt-in via ?include=summary.

Authorizations
APIKeyHeader ⌄
X-Api-Keystringheaderrequired
API key sent in the request header. JWT bearer auth is also accepted.
Path Parameters
projectIdOrNamestringrequired
The Ampersand project ID or project name.
operationIdstring · uuidrequired
The ID of the operation to diagnose.
Query Parameters
includeenumoptional
Set to summary to also return a natural-language narrative. Omitted by default to keep the response deterministic.
Response
categoryenumrequired
Failure class: auth permissions rate_limit provider_outage config unknown.
confidenceenumrequired
high · medium · low. Below threshold routes to unknown rather than guessing.
rootCauseobjectrequired
Structured explanation of the failure.
› Show child attributes
rootCause.codestring
Machine-readable failure code.
Example: PROVIDER_TOKEN_REVOKED
rootCause.summarystring
Plain-language explanation of the cause.
rootCause.providerErrorstring
Raw error string returned by the provider.
Example: INVALID_LOGIN_ATTEMPT
rootCause.scopeenum
single_customer · all_customers — whether the failure is isolated to this customer or affects everyone on the integration.
evidenceobject[]required
The signals justifying the diagnosis.
› Show child attributes
evidence[].typestring
The kind of signal.
Example: connection_status
evidence[].sourcestring
The API call the signal was derived from.
Example: GET /connections/conn-8f31
evidence[].detailstring
Human-readable description of the evidence.
remedyobjectrequired
Suggested fix, drawn from a fixed class→remedy mapping.
› Show child attributes
remedy.summarystring
Suggested fix in plain language.
remedy.actionsobject[]
Executable fix actions the dashboard renders as buttons.
› Show child attributes
actions[].kindstring
The action identifier.
Example: reauthenticate_connection
actions[].methodstring
HTTP method to call.
Example: PATCH
actions[].pathstring
Endpoint path to invoke the fix.
createTimestring · date-timerequired
When the diagnosis was computed (RFC-3339).

Query operations

Copy page ⌄
GET /projects/{projectIdOrName}/operations Try it
Proposed endpoint (Task 2). Sizing step — a project-wide, filterable query across every installation. Today's List operations is scoped to one installation.

Query operations across all installations in a project. Filter by status, provider, integration, customer, object, and time. Add groupBy to aggregate into counts and answer "is this one customer or all of them?" in a single call.

Authorizations
APIKeyHeader ⌄
X-Api-Keystringheaderrequired
API key sent in the request header. JWT bearer auth is also accepted.
Path Parameters
projectIdOrNamestringrequired
The Ampersand project ID or project name.
Query Parameters
statusenumoptional
success · failure · in_progress. Repeatable.
providerstringoptional
Filter by provider, e.g. netsuite.
integrationIdstringoptional
Filter to a single integration.
groupRefstringoptional
Filter to a single customer (group).
objectNamestringoptional
Filter by object, e.g. invoice.
since untilstring · date-timeoptional
createTime window (RFC-3339). Default lookback 7 days.
groupByenumoptional
groupRef · provider · status. Returns aggregated counts instead of a list.
pageSize pageTokeninteger / stringoptional
Pagination. pageSize max 1000, default 200.
Response
resultsobject[]
Operation objects (returned when groupBy is omitted).
› Show child attributes
results[].idstring
The operation ID.
Example: 60deaf48-3856-4a2b-bfd4-3de85125eca8
results[].groupRefstring
The customer this operation belongs to (denormalized).
Example: spreedly
results[].providerstring
The provider — denormalized so the fleet view renders without N+1 lookups.
Example: netsuite
results[].actionTypestring
read · write · subscribe.
results[].statusstring
success · failure · in_progress.
results[].resultstring
Human-readable outcome summary.
results[].createTimestring · date-time
When the operation ran (RFC-3339).
results[].metadataobject
Operation metadata.
› Show child attributes
metadata.objectsstring[]
Objects touched by the operation.
Example: ["invoice"]
metadata.retryobject
Retry attempts and lastAttempt timestamp.
paginationobject
Cursor pagination (list mode).
› Show child attributes
pagination.doneboolean
Whether this is the last page.
pagination.nextPageTokenstring
Cursor for the next page, if any.
groupsobject[]
Aggregated counts (returned when groupBy is set).
› Show child attributes
groups[].groupRefstring
The customer (group) reference.
Example: spreedly
groups[].groupNamestring
Display name of the customer.
groups[].failureCountinteger
Number of failing operations in the window.
Example: 6
groups[].latestFailurestring · date-time
Timestamp of the most recent failure.
totalFailingCustomersinteger
Count of customers with at least one failure (groupBy mode).
totalCustomersOnProviderinteger
Total customers on the filtered provider — powers the "3 of 41" ratio.
